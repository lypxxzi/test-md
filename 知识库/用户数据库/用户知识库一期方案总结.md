# 用户知识库一期方案总结

## 1、一期定位

一期先做一个轻量、可落地的 AI 知识库能力，核心目标是：

```text
用户可以上传 PDF / DOCX 文件到知识库，系统自动解析、切片、向量化，处理成功后可通过 AI 问答检索。
```

一期不做复杂的文档级 AI 检索权限，先按公司级知识库 Agent 落地。

## 2、核心概念

### 2.1 用户知识库

用户知识库是客户公司自己的知识库。

```text
归属：客户公司
内容来源：用户手动上传的文件
使用场景：基于公司自己的文件进行 AI 问答
检索范围：当前公司处理成功的知识库文件
```

### 2.2 运营知识库

运营知识库是平台级公共知识库。

```text
归属：平台方
内容来源：平台运营 / 实施 / 管理员维护
使用场景：回答系统怎么用、页面怎么操作、功能规则说明
检索范围：平台公共运营知识库
```

用户知识库和运营知识库管理入口建议分开，但底层 AI 入库、解析、切片、向量化、问答检索能力可以共用。

### 2.3 知识库文件

知识库文件是用户上传到知识库中的文件。

一期支持：

```text
PDF
Word docx
```

不支持：

```text
Excel
PPT
图片
压缩包
系统旧文件自动同步
业务系统附件自动入库
```

### 2.4 知识库 Agent

每个公司有一个用户知识库 Agent。

```text
用户知识库 Agent：检索当前公司的用户知识库文件
运营知识库 Agent / 能力：检索平台公共运营知识库
```

用户提问时需要选择：

```text
问公司文件：检索用户知识库
问系统使用：检索运营知识库
```

## 3、一期核心规则

### 3.1 文件进入知识库后自动处理

一期不单独设计“加入 AI 检索”按钮。

产品逻辑是：

```text
上传到知识库 = 系统自动建立 AI 检索能力
```

流程：

```text
上传文件
  ↓
系统自动解析
  ↓
切片
  ↓
向量化
  ↓
处理成功后可被 AI 检索
```

所以“加入检索”不是用户操作，而是系统处理成功后的结果。

### 3.2 文件状态

一期文件状态保留三类即可：

```text
处理中
成功
失败
```

含义：

```text
处理中：文件已上传，系统正在解析并建立 AI 检索索引，暂不可被 AI 检索。
成功：文件已处理完成，已进入知识库，可被 AI 检索。
失败：文件处理失败，未进入知识库，不能被 AI 检索。
```

建议在页面增加说明：

```text
AI 仅检索处理成功的文件，处理中和失败文件不会参与回答。
```

如果文件处理失败，需要展示失败原因，并支持删除或重新上传。

### 3.3 文件查看 / 下载权限

知识库文件可以设置查看和下载权限。

默认规则：

```text
默认仅上传人自己可查看 / 下载
可设置其他人查看 / 下载
```

但一期必须明确：

```text
文件查看 / 下载权限不等于 AI 检索权限。
```

一期 AI 检索规则是：

```text
文件处理成功进入用户知识库后，公司内具备 AI 使用权限的用户，都可能通过 AI 问答检索到文件内容。
```

因此，敏感文件不应上传到知识库。

### 3.4 删除文件

一期不做删除区、不做定时清理、不做恢复。

删除文件时直接删除相关数据：

```text
原文件
解析后内容
切片数据
向量数据
```

删除后，该文件不可再被 AI 检索。

删除权限建议：

```text
上传人可删除自己上传的文件
管理员可删除公司知识库文件
非上传人且非管理员不可删除
```

具体权限规则需要和开发确认。

## 4、一期业务流程

### 4.1 公司启用 AI 应用流程

```text
用户点击 AI 问答入口
  ↓
系统判断公司是否已启用 AI 应用
  ↓
未启用：判断当前用户是否管理员
  ↓
非管理员：提示联系管理员启用
  ↓
管理员：进入启用页面
  ↓
阅读免责声明和使用说明
  ↓
勾选同意
  ↓
点击“同意并启用”
  ↓
系统再次校验管理员身份
  ↓
启用成功
```

建议不要等用户读完免责声明后才校验管理员身份。

更合理的是：

```text
点击去启用时先校验是否管理员；
提交启用时后端再校验一次。
```

### 4.2 用户知识库文件上传流程

```text
进入用户知识库文件管理页
  ↓
点击新增文件
  ↓
选择文件分类
  ↓
上传 PDF / DOCX 文件
  ↓
设置查看 / 下载权限
  ↓
系统校验格式、大小、容量额度
  ↓
保存文件
  ↓
状态变为处理中
  ↓
系统自动解析、切片、向量化
  ↓
成功：状态变为成功，可被 AI 检索
  ↓
失败：状态变为失败，展示失败原因
```

### 4.3 用户知识库问答流程

```text
用户进入 AI 问答页
  ↓
选择“问公司文件”
  ↓
系统校验 AI 使用权限和额度
  ↓
用户输入问题
  ↓
系统把 company_id + question + conversation_id + user_id 传给 Agent
  ↓
Agent 在当前公司的用户知识库向量中检索
  ↓
召回相关片段
  ↓
调用大模型生成回答
  ↓
返回回答、引用来源、token 消耗
  ↓
系统保存会话和消息记录
```

### 4.4 运营知识库问答流程

```text
用户进入 AI 问答页
  ↓
选择“问系统使用”
  ↓
系统校验 AI 使用权限和额度
  ↓
用户输入系统操作类问题
  ↓
Agent 检索平台公共运营知识库
  ↓
调用大模型生成回答
  ↓
返回回答、引用来源、token 消耗
```

运营知识库是平台公共库，不按公司单独维护一套。

### 4.5 文件删除流程

```text
用户点击删除文件
  ↓
系统校验删除权限
  ↓
二次确认
  ↓
删除原文件
  ↓
删除解析内容
  ↓
删除切片
  ↓
删除向量数据
  ↓
文件从列表移除
```

## 5、一期页面 / 模块

### 5.1 AI 应用启用页

用于管理员启用 AI 知识库。

包含：

```text
功能说明
免责声明
使用说明
勾选同意
同意并启用按钮
```

需要记录：

```text
公司ID
启用人ID
启用人姓名
启用时间
免责声明版本号
使用说明版本号
启用状态
```

如果后续免责声明更新，需要考虑是否要求管理员重新确认。

### 5.2 用户知识库文件管理页

用于客户公司管理自己的知识库文件。

包含：

```text
新增文件
文件分类
文件查看 / 下载权限
文件状态
失败原因
删除文件
```

列表字段建议：

```text
文件名
分类
文件类型
文件大小
上传人
上传时间
处理状态
操作
```

### 5.3 用户知识库分类管理页

用于维护用户知识库文件分类。

包含：

```text
新增分类
编辑分类
删除分类
```

建议规则：

```text
已有文件使用的分类，不允许删除；
可以改名或停用。
```

### 5.4 AI 问答页

用户在这里进行 AI 问答。

需要选择问答类型：

```text
问公司文件
问系统使用
```

回答时需要展示引用来源，并标明来源类型：

```text
来源：用户知识库 / xxx.pdf
来源：运营知识库 / 操作手册
```

### 5.5 权益配置页

一期保留最小成本控制项：

```text
知识库文件数量上限
知识库存储容量上限
单个文件大小上限
公司每月 token 消耗上限
```

## 6、权益和 token 统计

### 6.1 额度统计原则

建议：

```text
用户知识库问答和运营知识库问答的 token 消耗，共同计入当前公司的 token 总额度。
```

原因：

```text
不管问哪个知识库，都是当前公司用户发起的模型调用，都会产生平台成本。
```

### 6.2 明细需要区分知识库类型

虽然额度一起扣，但明细需要分开记录：

```text
knowledge_type = USER / OPERATION
```

这样后续可以统计：

```text
公司本月总 token 消耗
用户知识库 token 消耗
运营知识库 token 消耗
```

### 6.3 token 统计方式

不建议每次通过聊天消息表连表实时 sum。

建议：

```text
明细要存
汇总要存
额度判断查汇总
```

推荐结构：

```text
消息表：记录单条消息 token 消耗，用于追溯
token 使用明细表：记录每次模型调用消耗
token 汇总表：按公司 / 用户 / 月 / 日维护累计消耗
```

提问流程：

```text
提问前查汇总额度
  ↓
额度足够则调用 Agent
  ↓
Agent 返回实际 token 消耗
  ↓
系统写入明细
  ↓
更新汇总
```

聊天记录删除不应影响已消耗 token 统计。

## 7、系统和 Agent 的边界

### 7.1 系统业务侧负责

```text
公司是否启用 AI
免责声明是否确认
谁能使用 AI
知识库文件上传
知识库分类
文件查看 / 下载权限
文件状态展示
文件删除
权益配置
会话记录展示
token / 次数统计
```

### 7.2 Agent 应用负责

```text
解析文件
切片
向量化
写入向量库
接收用户问题
问题向量化
检索相关片段
组装上下文
调用大模型
返回回答
返回引用来源
返回 token 消耗
```

### 7.3 对话时系统给 Agent 的数据

用户知识库问答：

```text
company_id
user_id
conversation_id
question
knowledge_type = USER
model_config，可选
max_context_token，可选
max_output_token，可选
```

运营知识库问答：

```text
user_id
company_id，用于额度统计
conversation_id
question
knowledge_type = OPERATION
model_config，可选
```

### 7.4 Agent 返回系统的数据

```text
answer
references
token_count
model_name
命中的 document_id / chunk_id
```

系统再保存到会话表和消息表中。

## 8、免责声明交互

### 8.1 入口

AI 问答入口可以在未启用时露出，但要明确状态：

```text
AI 知识库未启用
```

### 8.2 普通用户点击

普通用户点击后提示：

```text
当前公司尚未启用 AI 知识库，请联系公司管理员阅读并同意免责声明后启用。
```

不建议让普通用户进入免责声明阅读页后，最后才提示无权限。

### 8.3 管理员点击

管理员点击后进入启用页：

```text
功能说明
免责声明
使用说明
勾选同意
同意并启用
```

提交时后端再次校验管理员权限。

## 9、免责声明文案草稿

```text
AI 知识库使用免责声明

为保障企业数据安全，请在启用 AI 知识库前仔细阅读以下说明：

1、AI 知识库基于企业上传并处理成功的文件内容生成回答。AI 生成内容可能存在不准确、不完整或理解偏差，仅作为辅助参考，不应作为唯一决策依据。

2、文件上传至用户知识库并处理成功后，公司内具备 AI 使用权限的用户，可能通过 AI 问答检索到该文件中的相关内容。文件查看 / 下载权限不代表 AI 检索权限，请在上传前确认文件内容适合纳入公司 AI 知识库。

3、请勿上传涉及国家秘密、商业秘密、个人隐私、敏感数据、未授权资料、违法违规内容或其他不适合共享的文件。

4、用户应对上传文件的合法性、准确性、完整性及授权范围负责。因上传不当文件、错误使用 AI 回答或依据 AI 回答进行决策造成的风险，由使用方自行承担。

5、运营知识库内容用于辅助说明系统功能和操作流程，可能存在版本滞后或解释偏差，最终以系统实际页面、正式文档和平台通知为准。

6、平台将按照系统配置对文件数量、存储容量、单文件大小和 token 消耗进行限制。超出额度后，相关功能可能受到限制。

7、启用 AI 知识库即表示已阅读、理解并同意以上说明，并承诺按照企业内部数据安全要求使用本功能。
```

勾选文案：

```text
我已阅读并同意《AI 知识库使用免责声明》，并确认仅将适合公司内部 AI 检索的文件上传至用户知识库。
```

## 10、使用说明文案草稿

```text
AI 知识库使用说明

为提高 AI 问答准确性，建议上传文件时遵循以下规范：

1、建议上传内容清晰、结构完整的 PDF 或 Word（docx）文件。

2、建议一个文件只围绕一个主题，不要将多个无关主题合并到同一个文档中。

3、文档内容较长时，建议按章节、产品、流程或业务主题拆分为多个文件，便于 AI 更准确地检索。

4、建议使用清晰的标题、段落、编号和表格说明，避免大段无结构文本。

5、扫描版 PDF、图片型 PDF、低清晰度文件可能影响解析效果，不建议上传。

6、文件上传至知识库并处理成功后，公司内具备 AI 使用权限的用户可能通过问答获取相关内容，请谨慎上传敏感文件。

7、AI 回答基于知识库内容生成，可能存在偏差，请结合原文档和业务判断使用。

8、AI 仅检索处理成功的文件，处理中或处理失败的文件不会参与回答。
```

## 11、一期暂不做的内容

### 11.1 文档级 AI 检索权限

一期不做：

```text
某个文件只允许指定用户 / 分组通过 AI 检索
```

一期规则：

```text
文件处理成功进入用户知识库后，公司内具备 AI 使用权限的用户都可能检索到。
```

二期再考虑文档级 AI 检索权限。

### 11.2 系统旧文件入库

一期不做：

```text
系统历史文件自动同步
系统业务附件自动加入知识库
可加入知识库文件列表
```

一期只支持用户手动上传。

### 11.3 缓存文件区

一期不做缓存区。

上传到知识库后直接进入处理流程。

### 11.4 文件处理任务中心

一期不单独做文件处理任务中心。

文件处理状态在文件列表里展示即可。

二期在文件量大、批量上传多、需要集中排查时再考虑任务中心。

### 11.5 删除区 / 定时清理 / 恢复

一期删除即删除。

二期再考虑：

```text
删除区
保留时长
恢复
定时物理清理
```

### 11.6 更多文件格式

一期只支持：

```text
PDF
DOCX
```

二期再考虑：

```text
Excel
PPT
TXT
图片 OCR
扫描件 OCR
```

### 11.7 智能路由

一期用户手动选择：

```text
问公司文件
问系统使用
```

二期再考虑由 AI 自动判断应该检索用户知识库还是运营知识库。

## 12、二期方向

二期可以考虑：

```text
1、文档级 AI 检索权限
2、系统旧文件 / 业务附件入库
3、缓存文件区
4、文件处理任务中心
5、删除区和恢复机制
6、更多文件格式和 OCR
7、切片查看和重建
8、引用来源定位到页码 / 段落
9、相似度阈值和返回片段数配置
10、用户知识库 / 运营知识库智能路由
11、按知识库类型独立配置额度
```

## 13、一句话总结

```text
一期先做公司级用户知识库 Agent：客户手动上传 PDF / DOCX 文件，按分类管理，文件处理成功后可被公司内具备 AI 使用权限的用户问答检索；运营知识库作为平台公共库提供系统使用问答。两个知识库管理独立，底层 AI 入库和问答能力共用，token 额度合并扣减但按知识库类型记录明细。
```
