# 百事通返工与生产管理系统原型项目 (React Front-End)

这是一个使用 **Vite 5 + React 18 + Tailwind CSS + Lucide Icons** 构建的现代化、高度模块化的前端管理系统原型。

本系统根据化妆品生产和返工的核心实体设计规范（包括物料控制、BOM展开、MRP计算、先进先出FIFO领发料、中控工艺参数、批次有效期和采购回料等）编写，实现了**全业务流程的数据闭环流转联动**。

---

## 🛠️ 技术栈
- **构建工具**: [Vite 5.x](https://vitejs.dev/) (快如闪电的构建体验)
- **前端框架**: [React 18.x](https://react.dev/) (声明式、高效)
- **CSS 样式**: [Tailwind CSS 3.x](https://tailwindcss.com/) (现代实用类优先 CSS 框架)
- **图标组件**: [Lucide React](https://lucide.dev/) (像素级精美的开源图标库)
- **数据持久化**: `LocalStorage` (在不依赖后端的情况下，保存并模拟真实业务数据在各个模块的流转状态)

## 🚀 快速启动指南

### 1. 准备工作
确保你的电脑上安装了 [Node.js](https://nodejs.org/)（推荐使用 v18 或 v20 长期支持版）。

### 2. 安装依赖
打开你的终端（CMD、PowerShell 或 Git Bash），**进入 `frontend` 文件夹**，运行以下命令安装项目所需的依赖：

```bash
# 1. 进入前端项目目录
cd d:/a工作目录/百事通/frontend

# 2. 安装全部依赖
npm install
```

### 3. 运行本地开发服务器
运行以下命令启动本地调试服务：

```bash
npm run dev
```

启动成功后，Vite 将在控制台输出本地访问地址（默认是：`http://localhost:3000`）。
你的浏览器应该会**自动打开**这个页面。如果没有自动打开，手动在浏览器中输入 `http://localhost:3000` 即可！

## 🧭 核心演示流转链路 (如何进行业务演示)

本系统不仅是精美的 UI，所有的底层状态和操作全部打通，请按照以下闭环链路进行最完美的业务演示：

1. **录入待返工计划**：
   - 导航至 **“计划单 (返工计划)”** 菜单，点击右侧的 **“登记返工计划”**，选择配方并输入数量，提交后即生成并自动审批。
2. **下派生产单 & MRP 欠料展开**：
   - 在刚才创建的计划单上点击 **“下派生产单”**。
   - 输入下达数量后，系统会自动调用返工BOM的配方配比，并比对当前的 **“批次库存”**。
   - 如果发生欠料，系统会自动通过 MRP 计算缺口，并实时在 **“申购需求单 (欠料行)”** 中渲染出需要补充的甘油、内包材或外包材数量！
3. **申购转采购 & 供应商到货验收**：
   - 在 **“申购需求单 (欠料行)”** 中点击 **“一键申购”**。
   - 切换到 **“采购、回料与验收”** 页面，此时看到新生成的申购单。点击 **“向供应商订购下单”** 转为在途采购。
   - 点击 **“到货验收点收”**，输入实际验收数，系统会自动模拟质量质检和有效期计算，向 **“批次与库存管理”** 中追加一条全新的合格批次，库存状态实时更新！
4. **FIFO 自动发料与车间投产**：
   - 回到 **“生产单 (执行跟踪)”**。由于原料和包材已被采购点收充盈，原有的“物料严重短缺”提示解除。
   - 点击 **“自动FIFO发料配齐”**，仓库将以 **“先进先出”** 的原则自动锁定最早到货的原料批次并扣减对应数量，生产运行状态流转为 **“返工中”**。
5. **中控投料登记与完工入库**：
   - 点击 **“工序投料登记”**，可以记录和保存操作工在中控台的特定步骤与投料历史。
   - 完工后点击 **“包装完工入库”**，生产状态结案。系统自动向成品批次账册中注入带有 `FG`（返工）字头、自现在起 2 年失效保质期的全新成品批次。全流程至此完美闭环。
