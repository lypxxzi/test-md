# 中间表设计

## 整体架构（6 张表 = 日 3 + 月 3）

```
基础层（日粒度，ETL 直接生成）        汇总层（月粒度，从日表派生）
┌──────────────────────────────┐    ┌──────────────────────────────┐
│ dws_tenant_daily             │ →  │ dws_tenant_monthly           │ 企业
│ dws_customer_daily           │ →  │ dws_customer_monthly         │ 客户
│ dws_customer_product_daily   │ →  │ dws_customer_product_monthly │ 产品
└──────────────────────────────┘    └──────────────────────────────┘
```

**分层原则（遵循《设计思考记录.md》规范）：**

- **日表是唯一的事实基础**：所有指标先落到天粒度。流量型指标存「当日增量」，存量型指标存「当日快照」（规范1）。
- **月表从日表派生，不单独跑 ETL**：
  - 流量型字段（增量）→ 当月各天 `SUM`；
  - 存量型字段（快照）→ 取「当月最后一天」的值，**不能 SUM**。
- **滚动窗口（近30/90/180天）只能查日表**：`WHERE stat_date >= today - N`。月表无法精确表达滚动日窗口，仅用于规整的月/季/年趋势与排名。
- **每个字段都标注「增量 / 快照」**，这是建模规范，不是可选项。

> 字段类型说明：金额一律 `decimal(14,2)`（规范3）；日期日表用 `date`、月表用 `varchar(7)`（如 2025-06）。下文类型沿用通用写法，落到 ClickHouse 时 `bigint→Int64`、`int→Int32`、`tinyint→UInt8`、`varchar→String/LowCardinality(String)`。

# 一、基础层（日粒度）

## 表1：`dws_tenant_daily`（企业日汇总宽表）

**粒度：一行 = 一个租户 + 一天**

一个租户每天一行，是公司级别的每日总览。客户计数类指标只在企业粒度才有意义（客户粒度下每行就是一个客户，数不出来）。

### 维度字段（2列）

| 字段 | 类型 | 增量/快照 | 说明 |
|------|------|-----------|------|
| tenant_id | bigint | - | 租户ID（ORDER BY 第一列） |
| stat_date | date | - | 统计日期 |

### 客户计数字段（7列，均为快照）

| 字段 | 类型 | 增量/快照 | 说明 |
|------|------|-----------|------|
| total_customers | int | 快照 | 截至当日客户总数 |

# 二、汇总层（月粒度，从日表派生）

> 派生规则统一：**增量字段 `SUM`（当月各天求和）；快照字段取「当月最后一天」的值**。维度字段直接继承（SCD Type 1 直接覆盖，规范14）。月表字段与对应日表一一对应，仅把 `stat_date`(date) 换成 `stat_month`(varchar7)。

## 表4：`dws_tenant_monthly`（企业月度宽表）

**粒度：一行 = 一个租户 + 一个月**

字段与 `dws_tenant_daily` 完全对应（35列），派生方式：

| 字段组 | 派生方式 |
|--------|---------|
| total_customers / active_customers / inactive_customers / no_order_customers / reorder_customers | 取当月最后一天快照 |
| no_order_to_active_count / active_to_inactive_count | 当月各天 SUM（= 当月净转化人次） |
| order_count / order_amount / shipped_* / shipping_* / 按期交付增量 / 利润 / collected / receivable | 当月各天 SUM |
| in_production_* / unscheduled_* / overdue_count / pending / *_unallocated / total_uncollected | 取当月最后一天快照 |

---

## 表5：`dws_customer_monthly`（客户月度宽表）

**粒度：一行 = 一个租户 + 一个月 + 一个客户**

字段与 `dws_customer_daily` 完全对应（38列，`stat_date`→`stat_month`），派生方式：

| 字段组 | 派生方式 |
|--------|---------|
| is_active / has_order / is_reorder | 取当月最后一天快照 |

| order_count / order_amount / shipped_* / shipping_* / reorder_times / 按期交付增量 / 利润 / collected / receivable | 当月各天 SUM |
| in_production_* / unscheduled_* / overdue_count / pending / *_unallocated / total_uncollected | 取当月最后一天快照 |

# 四、派生（现算）指标口径

以下指标**不在表里存值**，查询时用已有字段现算（遵循规范3：宽表只存可加原始数值，比率/派生值现算）：

| 指标 | 口径/算法 | 用到的字段 |
|------|----------|-----------|
| 订单频次（平均下单周期） | **口径A**：周期天数 ÷ 周期订单数（如近30天下10单 = 3天/单） | 天数（查询窗口）÷ SUM(order_count) |
| 按期完成率 % | on_time_count ÷ total_product_count × 100 | on_time_count、total_product_count |
| 逾期率 % | overdue_count ÷ total_product_count × 100 | overdue_count、total_product_count |
| 预算利润率 % | budget_profit ÷ order_amount × 100 | budget_profit、order_amount |
| 实际利润率 % | actual_profit ÷ order_amount × 100（口径=已发货订单） | actual_profit、order_amount |
| 回款率 % | collected_amount ÷ order_amount × 100 | collected_amount、order_amount |
| 状态转化率 % | no_order_to_active_count ÷ total_customers × 100 等 | 转化计数 ÷ 基数 |

> 订单频次口径B（每客户相邻两单平均间隔再求平均）暂不实现——汇总表算不了，需要订单级明细。如后续需要再评估。
>
> 比率类算除法时用 `nullIf(分母, 0)` 防止除零。

# 五、需求映射（业务绩效需求 → 用哪张表）

| 需求 | 用表 | 说明 |
|------|------|------|
| 客户基础存量（总数/活跃/非活跃/未下单） | tenant_daily | 取最新一天快照，接近实时 |
| 活跃客户地域分布（省/市） | customer_daily | `is_active=1` + province/city 分组 |
| 近一年按月客户存量趋势 | tenant_monthly | 按 stat_month 取快照趋势 |
| 按月状态转化率 | tenant_monthly | no_order_to_active / active_to_inactive |
| 返单存量（近90/180天/近一年） | customer_daily | `stat_date >= today-N` 滚动窗口 |
| 高返单客户 TOP5（6个月/近一年）+按月趋势 | customer_daily / customer_monthly | reorder_times 求和排序 |
| 高订单量客户（近30/90/180天）TOP10 | customer_daily | 滚动窗口 SUM(order_count) 排序 |
| 高订单量客户（6个月/近一年）TOP5+趋势 | customer_monthly | 按月聚合 |
| 高订单金额客户（近30/90/180天）TOP10 | customer_daily | 滚动窗口 SUM(order_amount) 排序 |
| 高订单金额客户（6个月/近一年）TOP5+趋势 | customer_monthly | 按月聚合 |
| 全公司某产品排名/趋势 | customer_product_daily / monthly | 聚合掉 customer 维度即得企业×产品 |

# 六、查询示例

### 客户基础存量（实时，查日表取最新快照）

```sql
SELECT total_customers, active_customers, inactive_customers, no_order_customers
FROM dws_tenant_daily
WHERE tenant_id = 1 AND stat_date = (SELECT max(stat_date) FROM dws_tenant_daily WHERE tenant_id = 1)
```

### 高订单金额客户 TOP10（近30天滚动窗口，必须查日表）

```sql
SELECT customer_id, customer_name, SUM(order_amount) AS amount_30d
FROM dws_customer_daily
WHERE tenant_id = 1 AND stat_date >= today() - 30
GROUP BY customer_id, customer_name
ORDER BY amount_30d DESC
LIMIT 10
```

### 高返单客户 TOP5（近一年）+ 按月趋势

```sql
-- 近一年 TOP5
SELECT customer_id, customer_name, SUM(reorder_times) AS reorders
FROM dws_customer_daily
WHERE tenant_id = 1 AND stat_date >= today() - 365
GROUP BY customer_id, customer_name
ORDER BY reorders DESC
LIMIT 5

-- TOP5 客户按月返单趋势（用月表）
SELECT stat_month, customer_id, reorder_times
FROM dws_customer_monthly
WHERE tenant_id = 1 AND customer_id IN (...上面TOP5...)
ORDER BY stat_month
```

### 按月客户存量趋势（近一年，查月表快照）

```sql
SELECT stat_month, total_customers, active_customers, inactive_customers, no_order_customers
FROM dws_tenant_monthly
WHERE tenant_id = 1 AND stat_month BETWEEN '2024-07' AND '2025-06'
ORDER BY stat_month
```

### 按月状态转化率（近一年）

```sql
SELECT stat_month,
       no_order_to_active_count,
       active_to_inactive_count,
       no_order_to_active_count * 100.0 / nullIf(total_customers, 0) AS to_active_rate
FROM dws_tenant_monthly
WHERE tenant_id = 1 AND stat_month BETWEEN '2024-07' AND '2025-06'
ORDER BY stat_month
```

### 活跃客户地域分布（省/市）

```sql
SELECT province, city, count() AS active_cnt
FROM dws_customer_daily
WHERE tenant_id = 1 AND region_type = 'domestic'
  AND stat_date = (SELECT max(stat_date) FROM dws_customer_daily WHERE tenant_id = 1)
  AND is_active = 1
GROUP BY province, city
ORDER BY active_cnt DESC
```

### 全公司某产品近30天销量（聚合掉客户维度）

```sql
SELECT product_id, product_name, SUM(order_count) AS cnt_30d, SUM(order_amount) AS amt_30d
FROM dws_customer_product_daily
WHERE tenant_id = 1 AND stat_date >= today() - 30
GROUP BY product_id, product_name
ORDER BY amt_30d DESC
```

### 客户某产品按月趋势（查月表）

```sql
SELECT stat_month, order_count, order_amount
FROM dws_customer_product_monthly
WHERE tenant_id = 1 AND customer_id = 1001 AND product_id = 2001
ORDER BY stat_month
```
