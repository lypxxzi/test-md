## 2. 批次表

> 批次是库存管理的最小粒度，化妆品行业必须严格管理批次的生产日期、有效期和质检状态

| 字段名 | 字段类型 | 是否必填 | 说明 |
|---|---|---|---|
| ID | bigint | 是 | 主键，自增 |
| 批次号 | varchar | 是 | 业务批次号，如 BT20250601A001 |
| 物料编码 | varchar | 是 | FK→物料表 |
| 物料名称 | varchar | 是 | 系统带出 |
| 批次类型 | tinyint | 是 | 1-采购入库/2-生产入库/3-返工入库/4-调拨入库/5-盘盈 |
| 来源单据类型 | tinyint | 是 | 1-入库单/2-返工生产单/3-调拨单 |
| 来源单据编号 | varchar | 是 | 入库单号或生产单号 |
| 供应商批次号 | varchar | 否 | 采购时记录供应商原始批次 |
| 生产日期 | date | 是 | 实际生产日期 |
| 有效期至 | date | 是 | 化妆品关键字段，用于效期预警 |
| 入库时间 | datetime | 是 | 首次入库时间 |
| 入库数量 | decimal | 是 | 初始入库数量 |
| 剩余数量 | decimal | 是 | 当前可用库存（实时计算或触发更新）|
| 质检状态 | tinyint | 是 | 0-待检/1-合格/2-不合格/3-免检 |
| 状态 | tinyint | 是 | 0-正常/1-冻结/2-过期/3-报废 |
| 备注 | varchar | 否 | |

> **批次号生成规则建议**：
> - 采购批次：`BT` + 入库日期(YYYYMMDD) + 物料简码(2位) + 流水号(3位)
> - 生产批次：`SC` + 生产日期(YYYYMMDD) + 产线代码(1位) + 流水号(3位)
> - 返工批次：`FG` + 返工日期(YYYYMMDD) + 原批次号后4位 + 流水号(2位)

> **关键业务规则**：
> - 先进先出(FIFO)：出库时优先选择最早生产日期且未过期的批次
> - 效期预警：有效期前30天自动预警，过期自动冻结
> - 质检锁定：质检状态=待检时，批次状态自动=冻结，不可出库

## 12. 投料子单（工序投料明细）

> 投料主单的子表，记录每个工序实际投入的物料明细

| 字段名 | 字段类型 | 是否必填 | 说明 |
|---|---|---|---|
| ID | bigint | 是 | 主键，自增 |
| 投料主单ID | bigint | 是 | FK→投料主单 |
| 行号 | int | 是 | |
| 物料编码 | varchar | 是 | FK→物料表 |
| 物料名称 | varchar | 是 | 系统带出 |
| 规格型号 | varchar | 否 | 系统带出 |
| 单位 | varchar | 是 | 系统带出 |
| 关联BOM步骤投料明细ID | bigint | 是 | FK→BOM步骤投料明细，带出计划投料比 |
| 计划投料数量 | decimal | 是 | 系统计算：BOM投料比 × 生产数量 |
| 实际投料数量 | decimal | 是 | 实际投入数量 |
| 批次ID | bigint | 是 | FK→批次表，关联实际使用的批次 |
| 库位 | varchar | 否 | 物料来源库位 |
| 投料模式 | tinyint | 是 | 1-直接投料/2-备料后投料 |
| 关联备料子单ID | bigint | 否 | FK→备料子单，备料后投料时必填 |
| 备注 | varchar | 否 | |

> **备料与投料关系处理**：
> - **多次备料**：一个工序可创建多个备料单（如分批次称量），每个备料单含部分物料
> - **模式1-直接投料**：无备料记录，投料子单直接录入物料批次和数量
> - **模式2-备料后投料**：投料子单引用备料子单ID，自动带出批次和数量，可修改实际投料数量（允许损耗差异）
> - **混合模式**：同工序内部分物料走备料流程、部分直接投料（如主料先称量、辅料直接拿）
> - **校验规则**：
>   - 直接投料：校验库存可用量
>   - 备料后投料：校验备料子单所属备料主单状态为"已完成"，且该备料子单未被引用
>   - 子单级别一对一：一个备料子单只能被一次投料引用

> **差异说明**：实际投料数量可能与计划不同（损耗、调整等），记录真实投料情况用于成本核算

| 实际称量数量 | decimal | 是 | 实际称量的数量 |

> **备料子单生命周期**：
> - 备料主单完成后，其下明细可被投料子单引用
> - 被引用后锁定不可修改
> - 子单级别一对一：一个备料子单只能被一个投料子单引用

> **备料 vs 投料区别**：
> - 备料：准备阶段，称量好物料放在一边待用
> - 投料：实际投入设备开始生产
> - 两者数量可能有差异（备多了、损耗等）
