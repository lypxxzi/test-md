# 化妆品成品返工需求调研文档（V2.0）

> **范围限定**：仅考虑已入库成品的返工场景，通过新建返工生产单完成返工闭环。
> 后续需将返工数据推送金蝶云星空，设计时需考虑兼容性。
> 优先以化妆品成品出发，完成整个需求的了解、调研、分析、业务流程、实体抽象及整体逻辑闭环。

---

## 一、产品结构与生产主流程

### 1.1 产品层级结构

```
产品（成品）= 中间品 + 外包材
  └── 中间品 = 配方产品 + 内包材
        └── 配方产品 = 配方 + 产品标准
              └── 配方 = 原料 + 工艺流程
```

### 1.2 完整生产主流程

生产主流程按产品层级分为三个阶段：

```
【配方产品阶段】                  【中间品阶段】                【成品阶段】
投入：原料                       投入：配方产品 + 内包材        投入：中间品 + 外包材
──────────────                 ──────────────              ──────────────

1. 工作人员登记                       │                          │
2. 生产前消毒                         │                          │
3. 生产配制操作流程 ──→ 配方产品 ──→  6. 内包装（灌装）──→ 中间品 ──→  8. 外包装（贴标/装盒）
4. 出锅检验（锅检）                    7. [内包检验]                9. 成品质检
5. 物料平衡/清场消毒                                             10. 入库
                                                              11. 统计工时
```

| 产品层级 | 对应生产阶段 | 涉及物料 | 涉及步骤 |
|---------|------------|---------|---------|
| 配方产品 | 配制阶段 | 原料 | 步骤1-5 |
| 中间品 | 灌装阶段 | 配方产品 + 内包材（瓶、管、袋、盖） | 步骤6-7 |
| 成品 | 包装阶段 | 中间品 + 外包材（标签、彩盒、说明书） | 步骤8-9 |

### 1.3 两种"流程"的区分

| | 生产主流程（管控框架） | 生产配制操作流程（工艺路线） |
|---|---|---|
| **本质** | 确保每步都执行到位 | 指导工人具体怎么操作 |
| **是否固定** | 基本固定，所有产品都走 | 每个产品/配方不同 |
| **举例** | "第4步：出锅检验"——做了就打勾 | "第2步：加入XX原料5kg，升温至80°C，搅拌30分钟" |
| **金蝶对应** | 更接近"业务流程" | 工艺路线（工序 + 工序参数） |

## 二、已入库成品触发返工的场景

**前提**：仅考虑产品已经完成入库（成品仓）的情况。

| # | 场景 | 触发方式 | 典型原因 |
|---|------|---------|---------|
| 1 | 留样/稳定性检测不合格 | 入库后留样跟踪发现问题 | pH漂移、乳化不稳定、析出 |
| 2 | 客户验货不通过 | 客户到仓验货或收到样品后拒收 | 外观/气味/颜色不符客户标准 |
| 3 | 标签/包装变更 | 法规更新或客户要求变更标签 | 成分表更新、备案编号变更、客户换标 |
| 4 | 临期翻包 | 产品临近效期需要重新包装 | 效期标识更新、促销装重组 |
| 5 | 发货前抽检不合格 | 出库前放行抽检发现问题 | 仓储条件导致的质量变化 |
| 6 | 退货返工 | 客户退货回仓后需返工再利用 | 外包装破损但内容物完好 |

## 三、返工方案设计——一张返工生产单搞定

### 3.1 核心设计原则

返工采用**两层单据结构**：

```
返工单（决策层）
  └── 可包含多个产品、多个批次
  └── 记录：为什么返工、涉及哪些批次、由谁审批

返工生产单（执行层）
  └── 一张单对应一个产品一个批次
  └── 记录：怎么执行、BOM、工艺路线、拆解记录、质检结果
```

**分层原因**：一次返工事件（如法规变更）可能涉及仓库中多个产品的多个批次，需要在决策层统一管理；但每个产品/批次的执行方案（BOM、工艺、拆解量）各不相同，需要在执行层独立管控。

返工生产单 = 在现有生产单基础上增加**类型=返工**，并放开BOM和工艺路线的编辑权限。

### 3.2 返工生产单与普通生产单的区别

| | 普通生产单 | 返工生产单 |
|---|---|---|
| **触发** | 指令单/订单 | 质检不合格/客户要求 |
| **BOM** | 系统按产品自动带出，不可修改 | 基于原BOM复制，**可编辑** |
| **投入物料** | 全部从仓库领新料 | 一部分是不合格品出库（回收物料），一部分是新领料 |
| **工艺路线** | 按产品配置，不可修改 | 基于原工艺复制，**可编辑** |
| **起始步骤** | 从头开始（步骤1） | 根据返工类型从中间某步开始 |
| **报损记录** | 无 | 有拆解报损 |
| **关联单据** | 关联指令单 | 关联原批次号、质检单 |

### 3.3 返工类型与执行路径

| 返工类型 | 拆解到哪一层 | 从哪个阶段重新走 | 需重新经过的质检点 |
|---------|------------|----------------|-----------------|
| **外包装返工**（换标/换盒） | 拆外包材，中间品不动 | 步骤8：外包装 | 成品质检 |
| **内包装返工**（重灌） | 拆内包材，倒出料体 | 步骤6：灌装 | 内包检 → 外包装 → 成品质检 |
| **配制返工**（回锅调配） | 全部拆解到料体 | 步骤3：配制 | 锅检 → 灌装 → 内包检 → 外包装 → 成品质检 |
| **微生物超标** | **不可返工** | — | 系统拦截，只能报废 |

### 3.4 返工BOM处理——基于原BOM复制+手动调整

返工BOM无法提前预设（每次返工原因不同，物料需求不同），采用**复制+调整**策略：

**操作流程**：

```
1. 创建返工生产单，选择产品和返工类型
2. 系统自动复制原产品BOM → 生成返工BOM草稿
3. 操作人员根据返工方案手动调整：
   - 删除不需要的物料行
   - 增加调整用物料（如柠檬酸调pH）
   - 修改数量
   - 标记物料来源（回收 or 新领）
4. 确认后，按此BOM执行领料
```

**BOM物料来源标记**：

| 物料来源 | 说明 | 是否生成领料需求 |
|---------|------|----------------|
| 返工回收 | 拆解回收的料体/可复用物料 | 否，从返工出库直接转入 |
| 仓库领料 | 新追加的物料（原料/包材） | 是，走正常领料流程 |

**示例——回锅调配的返工BOM**：

```
原始BOM                              返工BOM（调整后）
─────────                            ─────────
原料A  10kg                          回收料体  14.5kg    [来源=返工回收]
原料B   5kg                          柠檬酸    0.3kg     [来源=仓库领料]  ← 新增
内包材-瓶  1000个                     新瓶     1000个     [来源=仓库领料]
内包材-盖  1000个                     新盖     1000个     [来源=仓库领料]
外包材-标签  1000张                   新标签   1000张     [来源=仓库领料]
外包材-彩盒  1000个                   新彩盒   1000个     [来源=仓库领料]
─────────────────                    ─────────────────
产出：成品 1000瓶                     产出：成品 950瓶（考虑损耗）
```

### 3.5 返工单设计——触发来源与批次选择

#### 3.5.1 五种触发入口

不同触发场景下，返工单的创建入口和批次选择方式不同：

| 触发来源 | 创建入口 | 批次选择方式 | 典型场景 |
|---------|---------|------------|---------|
| **质检单** | 质检单页面点"发起返工" | 系统自动带出产品+批次，无需手动选 | 留样检测pH超标 |
| **客诉/退货单** | 客诉单页面点"发起返工" | 系统自动带出退货批次 | 客户反馈标签印刷错误 |
| **法规变更** | 手动新建返工单 | 按产品筛选 → 列出所有在库批次 → 勾选多个 | 新规要求更换成分标注 |
| **临期翻包** | 手动新建返工单 | 按有效期筛选（如"60天内到期"）→ 列出临期批次 → 勾选 | 临近效期需更换包装 |
| **手动发起** | 手动新建返工单 | 搜索产品编码/名称 → 选批次 → 手动添加 | 其他零散情况 |

#### 3.5.2 批次选择时的系统校验

选择批次加入返工单时，系统自动执行以下校验：

| 校验项 | 规则 | 不通过时的处理 |
|-------|------|-------------|
| **重复返工** | 该批次是否已存在"进行中"的返工单/返工生产单 | **拦截**，提示"该批次已在返工单 RW-xxx 中，请勿重复发起" |
| **库存存在** | 该批次在成品仓是否有可用库存（非冻结） | **拦截**，提示"该批次无可用库存" |
| **微生物超标** | 关联质检单中是否存在微生物超标不合格项 | **拦截**，提示"微生物超标不可返工，请走报废流程" |
| **返工次数** | 该批次历史返工次数是否已达上限（默认2次） | **警告**（不拦截），需额外审批人确认才可继续 |

### 3.6 返工工艺路线处理——基于原工艺复制+手动调整

与BOM同理，采用**复制+调整**策略：

```
1. 系统自动复制原产品工艺路线 → 生成返工工艺草稿
2. 操作人员根据返工方案调整：
   - 删除不需要执行的步骤
   - 修改工艺参数（如温度、时间）
   - 增加新的操作步骤（如"加入柠檬酸调pH"）
3. 确认后，按此工艺路线指导车间操作
```

**示例——回锅调配的工艺路线调整**：

```
原始工艺路线                          返工工艺路线（调整后）
─────────                            ─────────
1. 投料A → 升温80°C                  1. 投入回收料体 → 升温60°C
2. 投料B → 乳化30分钟                2. 加入柠檬酸 → 搅拌15分钟    ← 新增
3. 降温至40°C → 出锅                  3. 测pH → 合格后降温出锅
```

**补充**：返工方案中同时提供一个**富文本"返工方案说明"字段**，由QA/技术人员填写。简单返工（换标/换盒）可能不需要调整工艺路线，只需方案说明即可。

## 四、返工生产单完整流程

### 4.1 不合格品出库的物料层级分析

仓库中存储的永远是**成品**，"不合格品出库"出的也永远是成品。但拆解后回收物的层级因返工类型而不同：

| 返工类型 | 出库的是什么 | 拆解后回收的"好东西" | 回收物在BOM中的角色 |
|---------|------------|------------------|-----------------|
| **外包装返工** | 成品 | **中间产品**（灌装封口完好的瓶子）| BOM输入：中间产品（来源=返工回收）+ 新外包材 |
| **内包装返工** | 成品 | **料体**（需复检确认是否合格）| BOM输入：料体（来源=返工回收）+ 新内包材 + 新外包材 |
| **配制返工** | 成品 | **有问题的料体**（需要调配修正）| BOM输入：料体（来源=返工回收，需调配）+ 调整用原料 + 新内包材 + 新外包材 |

> **注意**：返工BOM中"来源=返工回收"的物料编码，外包装返工对应中间产品编码，内包装/配制返工对应料体编码，需与系统中已有的多层BOM编码体系保持一致。

### 4.2 状态流转

```
[草稿] → [审批中] → [已审批] → [待出库] → [已出库/待领料] → [执行中] → [待质检] → [已完成/入库]
                       │                                                       │
                       ▼                                                       ▼
                    [已驳回]                                              [质检不合格]
                                                                              │
                                                                      ┌───────┴───────┐
                                                                      ▼               ▼
                                                                  [再次返工]        [报废]
```

### 4.3 各步骤详细说明

| 步骤 | 操作人 | 系统动作 |
|------|--------|---------|
| **1. 创建返工生产单** | QA/生产主管 | 填写基本信息（原批次、返工原因、返工类型）；系统自动复制原BOM和工艺路线；操作人员调整BOM和工艺（含预估回收率）；填写返工方案说明 |
| **2. 审批** | QA负责人 | 审核返工方案、BOM、工艺路线是否可行 |
| **3. 不合格品出库→入返工待处理仓** | 仓库 | 审批通过后，成品从**成品仓出库**，入到**返工待处理仓**（逻辑库位）；库存状态由"正常"变为"待返工"；系统冻结该批次，防止被其他单据占用 |
| **4. 新物料领料** | 车间/仓库 | 按返工BOM中"来源=仓库领料"的物料生成领料需求；从原料仓/包材仓领取 |
| **5. 拆包（生产工序）** | 车间 | **拆包作为生产工序执行**（不是先拆再开单）；从返工待处理仓领出成品 → 拆解 → 拆解结果分两路：① 回收物（中间产品/料体）**入返工暂存仓**；② 不可用包材残料**入报废待处理区** |
| **5a. 回收物入库** | 仓库/系统 | 拆解回收的料体或中间产品，以新批次号入到**返工暂存仓**，并自动与本返工生产单关联（预留给本单，不可被其他单据占用） |
| **6. 回收物领料** | 车间 | 从返工暂存仓领出关联的回收批次，系统按返工生产单号自动匹配对应批次，操作员无需手动选 |
| **7. 生产执行** | 车间 | 按返工工艺路线和主流程执行，从返工类型对应的起始步骤开始走 |
| **8. 过程质检** | QC | 走到质检节点时执行检验（锅检、内包检等），复用现有质检模块 |
| **9. 成品质检** | QC | 全项目复检，标注"返工批次，第X次返工"，标准与原产品一致 |
| **10a. 合格→入库** | 仓库 | 返工成品从生产区入成品仓，使用新批次号（原批次号-R1） |
| **10b. 不合格→处置** | QA | 判定报废或再次返工；达到次数上限仍不合格→强制报废（次数可配置） |

### 4.4 不同返工类型的执行路径

**简单返工（换标/换盒）**：

```
出库 → 拆外包 → 重新外包装 → 成品质检 → 入库
           └→ 报损旧外包材
```

**中度返工（重灌）**：

```
出库 → 拆外包 → 拆内包/倒料 → 重新灌装 → 内包检 → 外包装 → 成品质检 → 入库
           └→ 报损旧包材       └→ 回收料体
```

**复杂返工（回锅调配）**：

```
出库 → 全拆解 → 料体回锅调配 → 锅检 → 灌装 → 内包检 → 外包装 → 成品质检 → 入库
    └→ 报损全部包材  └→ 加入调整用原料
```

## 五、角色定义

### 5.1 核心角色

| 角色 | 职责 | 关键决策权 |
|------|------|-----------|
| QC（质量控制） | 执行检验、出具不合格报告、执行返工过程中各环节质检和最终复检 | 判定合格/不合格 |
| QA（质量保证） | 主导评审、审批返工生产单（含方案/BOM/工艺）、批记录审核、最终放行 | 是否可返工、方案审批、成品放行 |
| 生产主管 | 参与评审、安排返工排产、监督车间执行 | 返工排期、人员安排 |
| 技术/研发 | 提供返工技术方案（回锅配方调整、工艺路线调整建议） | 技术可行性判定 |
| 仓库管理员 | 不合格品出库、追加物料出库、返工成品入库 | 无决策权，执行角色 |
| 车间操作人员 | 执行拆解、按工艺路线操作、填写批记录 | 无决策权，执行角色 |

### 5.2 关联角色

| 角色 | 参与场景 |
|------|---------|
| 采购 | 返工追加物料库存不足时触发紧急采购 |
| 销售 | 返工影响订单交期时通知客户 |
| 客户 | 让步放行场景下确认是否接受 |
| 财务 | 返工成本核算与归集 |

### 5.3 审批链路

```
触发事件（质检不合格/客户要求/法规变更）
  → QA/生产主管创建返工生产单（含方案、BOM、工艺路线）
    → QA负责人审批
      → 仓库出库 + 领料
        → 车间执行拆解和返工生产
          → QC各环节质检 + 最终成品质检
            → QA最终放行/不放行
```

## 六、基础数据支持

### 6.1 需要新增的基础数据

| 基础数据 | 数据项 | 说明 |
|---------|--------|------|
| 返工原因字典 | pH超标、粘度不合格、色差超标、灌装量偏差、封口不严、标签歪斜、外盒破损、客户换标、法规变更… | 下拉选择 |
| 返工类型字典 | 外包装返工、内包装返工、配制返工 | 决定执行路径和起始步骤 |
| 返工次数上限配置 | 按产品类别设置最大返工次数（默认2次） | 超限强制报废 |
| 处置方式字典 | 合格入库、报废、让步放行 | 返工结果的处置 |

### 6.2 现有基础数据的复用

| 现有数据 | 与返工的关联 |
|---------|------------|
| 产品BOM | 创建返工生产单时自动复制，作为返工BOM的编辑基础 |
| 工艺路线 | 创建返工生产单时自动复制，作为返工工艺的编辑基础 |
| 质检标准 | 返工各环节质检（锅检、内包检、成品质检）复用现有标准 |
| 物料主数据 | 返工BOM中物料编码需与现有物料主数据一致 |
| 仓库/库位 | 涉及成品仓出库、原料仓/包材仓领料、成品仓入库 |
| 生产主流程 | 返工复用现有主流程各质检节点 |

### 6.3 需新增的返工专用库位

返工流程引入三个**逻辑库位**（不是新建物理仓库，在金蝶现有仓库下新建库位即可）：

| 库位名称 | 存放内容 | 流入 | 流出 | 库存状态 |
|---------|---------|------|------|---------|
| **返工待处理仓** | 审批通过后待拆包的不合格成品 | 成品仓出库转入 | 拆包工序领出 | 冻结，不可被其他单据占用 |
| **返工暂存仓** | 拆包后回收的中间产品或料体 | 拆包工序入库 | 返工生产领料 | 预留，仅本返工生产单可领用 |
| **报废待处理区** | 拆包产生的不可用包材残料 | 拆包工序入库 | 定期报废处置单 | 冻结 |

> 三个库位的设计好处：不合格品在任意时刻都有明确的库位归属，审计可查；回收物料通过预留机制与返工生产单绑定，防止被其他单据错误领用。

## 七、化妆品返工相关法规

### 7.1 核心法规依据

| 法规 | 相关条款 | 对返工的要求 |
|------|---------|------------|
| 《化妆品生产质量管理规范》(2022) | 第44-46条 | 不合格品应隔离存放、标识明确；返工需经质量授权人批准 |
| 《化妆品监督管理条例》(2021) | 第29条 | 化妆品注册人/备案人应建立质量管理体系 |
| 《化妆品生产经营监督管理办法》 | 第32条 | 企业应当建立不合格品管理制度 |
| GB/T 32670-2016 | 第8、9章 | 化妆品生产企业良好生产规范 |
| ISO 22716:2007 | 第9章 | 不合格品处理、返工需有文件化程序 |

### 7.2 法规对系统设计的硬性约束

1. **必须有书面批准**：返工方案需质量负责人审批，系统需电子审批流
2. **批记录完整性**：返工过程必须纳入批生产记录，不能断链
3. **标识管理**：返工中物料/半成品须有明确标识，系统体现为批次状态标记
4. **隔离管理**：不合格品须物理隔离或系统冻结
5. **时限要求**：不合格品不能无限期搁置，需超时预警
6. **微生物超标限制**：微生物超标通常不允许返工，系统需校验拦截
7. **追溯要求**：返工品最终去向须100%可追溯
8. **记录保存**：记录保存期限不少于产品使用期限届满后1年

## 八、返工流程可追溯性设计

### 8.1 追溯链路模型

```
原成品批次号
  └── 返工生产单号
        ├── 基本信息（返工原因、关联质检单、审批记录）
        ├── 返工BOM（基于原BOM调整后的物料清单，含物料来源标记）
        ├── 返工工艺路线（基于原工艺调整后的操作步骤）
        ├── 出库记录（不合格品从成品仓出库）
        ├── 拆解/报损记录（回收物料量、报废物料量）
        ├── 领料记录（新追加物料的批次号、数量）
        ├── 生产执行批记录（各步骤操作记录）
        ├── 各环节质检报告（锅检、内包检、成品质检）
        └── 入库记录（返工成品入库，新批次号）
```

### 8.2 关键追溯维度

| 追溯维度 | 实现方式 |
|---------|---------|
| **正向追溯**：原料→成品→返工记录 | 返工生产单关联原批次号，返工后批次号带后缀 |
| **反向追溯**：成品→追加物料来源 | 领料记录关联物料批次号 |
| **状态追溯**：批次在任意时间点的状态 | 批次状态机记录每次变更 |
| **决策追溯**：谁批准、基于什么判断 | 审批流日志 |
| **操作追溯**：返工过程做了什么 | 批记录逐步记录 |

### 8.3 批次号策略

#### 两类批次号的处理规则

| 场景 | 是否用旧批次号 | 规则 |
|------|-------------|------|
| **拆解回收物**（料体、中间品） | ❌ 不用 | 系统分配临时批次号，绑定返工生产单，仅本单可领用，不对外暴露 |
| **返工完成再入库的成品** | ❌ 不用 | 使用新批次号：原批次号 + 返工后缀，如 `CPZ-20250525-001-R1` |

#### 为什么不能沿用旧批次号

1. **可追溯性要求**：返工品经过了不同的生产过程（不同BOM、可能追加了新物料），与原批次的工艺链不同，同一批次号映射两段不同生产记录会造成追溯混乱
2. **标识管理要求**：化妆品GMP（2022）第44-46条要求不合格品有明确标识，返工品须与原批次可区分
3. **ISO 22716:2007 第9章**：要求返工品批记录引用原批次，并有独立标识
4. **监管检查实践**：检查人员普遍期望返工品使用独立批次号（参照药品GMP实践）

#### 特殊情况：纯外包装返工（只换标签）

虽然产品本体未变，仍**推荐使用新批次号**：
- 换标签本身就要重新打印，顺带更新为 `-R1` 无额外成本
- 避免同一批次号下出现多段不同操作记录
- 追溯时任何人看到 `-R1` 即知有返工历史，不必翻查批记录

#### 批次号格式

- 第一次返工：`原批次号-R1`，如 `CPZ-20250525-001-R1`
- 第二次返工：`CPZ-20250525-001-R2`（基于原始批次号续编，不叠加为 `-R1-R2`）
- 返工暂存仓中的回收物：`原批次号-RW`（内部追踪用，不入最终批记录）

## 九、返工流程成本归集

### 9.1 成本结构：返工总成本 = 损失成本 + 新增投入成本

| 大类 | 成本项 | 含义 | 计算公式 | 数据来源（返工生产单中的子表） |
|------|--------|------|---------|---------------------------|
| **损失成本** | 拆解报废物料 | 拆解过程中不可复用的包材/原料 | 报废数量 × 物料单价 | 拆解/报损记录子表（类型=报损） |
| **损失成本** | 料体残留损耗 | 倒料过程中瓶壁残留等不可回收部分 | 损耗数量 × 料体单价 | 拆解/报损记录子表（类型=报损） |
| **新增投入** | 追加原料成本 | 返工过程中新领用的原料（如柠檬酸调pH） | 实际领用量 × 物料单价 | 返工BOM子表（来源=仓库领料，物料类型=原料） |
| **新增投入** | 新包材成本 | 返工过程中新领用的内/外包材 | 实际领用量 × 物料单价 | 返工BOM子表（来源=仓库领料，物料类型=包材） |
| **新增投入** | 人工成本 | 拆解+配制+灌装+包装等各环节人工 | 各步骤工时合计 × 工时费率 | 批记录中各步骤的操作时间差 |
| **新增投入** | 设备占用成本 | 配制锅、灌装机等设备占用 | 各设备使用时长 × 设备费率 | 批记录中记录的设备编号和时长 |
| **新增投入** | 质检成本 | 各环节复检费用（料体复检/锅检/内包检/成品质检） | 质检次数 × 单次检测费 | 关联的质检单数量 |

### 9.2 成本计算示例：配制返工（回锅调配）1000瓶100ml面霜

#### 损失成本（拆解报废）

| 报废物料 | 数量 | 单价 | 金额 | 说明 |
|---------|------|------|------|------|
| PET瓶 | 1000个 | 0.35元 | 350元 | 拆解后不可复用 |
| 螺旋盖 | 1000个 | 0.12元 | 120元 | 同上 |
| 产品标签 | 1000张 | 0.08元 | 80元 | 撕标损毁 |
| 彩盒 | 1000个 | 0.25元 | 250元 | 拆包破损 |
| 料体残留损耗 | 0.5kg | 80元/kg | 40元 | 瓶壁残留约5%不可回收 |
| **小计** | | | **840元** | |

#### 新增投入成本

| 成本项 | 明细 | 金额 | 说明 |
|--------|------|------|------|
| 追加原料 | 柠檬酸 0.3kg × 50元/kg | 15元 | 调pH用 |
| 新包材 | 新瓶1000×0.35 + 新盖1000×0.12 + 新标签1000×0.08 + 新彩盒1000×0.25 | 800元 | 全部换新 |
| 人工 | 拆解2h + 配制1h + 灌装2h + 包装2h = 7工时 × 35元/时 | 245元 | |
| 设备 | 配制锅1h×20元 + 灌装机2h×30元 | 80元 | |
| 质检 | 料体复检 + 锅检 + 内包检 + 成品质检 = 4次 × 50元 | 200元 | |
| **小计** | | **1340元** | |

#### 合计

| | 金额 |
|---|------|
| 损失成本（报废） | 840元 |
| 新增投入成本 | 1340元 |
| **返工总成本** | **2180元** |

> 注：不同返工类型的成本差异很大。外包装返工（换标）可能只有几百元（旧标签报废+新标签+人工），而配制返工涉及全部拆解和重新走完整流程，成本最高。

### 9.3 需新增的费率基础数据

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 工时费率 | 可按车间或工种区分 | 普工35元/时、技术工50元/时 |
| 设备费率 | 按设备类型配置 | 配制锅20元/时、灌装机30元/时 |
| 单次质检费用 | 按检测类型区分 | 内检50元/次、外送检200元/次 |

### 9.4 成本归属原则

- 返工成本归集到**原批次**
- 区分展示：正常生产成本 vs 返工附加成本
- 返工成本不摊入其他正常批次

### 9.5 成本报表需求

| 报表 | 维度 | 用途 |
|------|------|------|
| 单批次返工成本明细 | 按成本项拆分（损失/新增投入） | 单批次分析 |
| 月度/季度返工总成本 | 按时间汇总 | 管理层决策 |
| 按返工类型分类成本 | 外包装/内包装/配制 | 不同类型的成本对比 |
| 按返工原因分类成本 | 按返工原因 | 找出高成本问题类型 |
| 返工率统计 | 返工批次数/总批次数 | KPI考核 |
| 产品维度返工分析 | 按产品/类别 | 识别高风险产品 |

## 十二、金蝶云星空对接设计（预留）

### 12.1 对接原则

- 当前阶段金蝶推送暂不做完善，**预留接口字段即可**
- 返工生产单推送金蝶时映射为**金蝶生产订单（类型=返工）**
- 基础数据编码（物料、仓库、单位）需与金蝶保持一致

### 12.2 推送单据映射

| 返工生产单动作 | 推送金蝶的单据 | 推送时机 |
|--------------|-------------|---------|
| 审批通过 | 生产订单（类型=返工） | 审批通过时 |
| 领料 | 生产领料单 | 实际领料时 |
| 返工完成入库 | 产成品入库单 | 质检合格入库时 |

### 12.3 返工生产单需预留的金蝶对接字段

| 字段 | 说明 |
|------|------|
| 金蝶原生产订单号 | 推送时作为源单引用 |
| 金蝶BOM版本号 | 告诉金蝶用哪个BOM（后续返工BOM需同步到金蝶） |
| 金蝶推送状态 | 未推送/推送中/已推送/推送失败 |
| 金蝶返回单据号 | 推送成功后回写 |

### 12.4 金蝶返工BOM的处理策略

| 方案 | 做法 | 适用场景 |
|------|------|---------|
| A. 金蝶侧预设返工BOM | 每个产品在金蝶维护固定返工BOM版本 | 返工场景相对固定时（推荐初期使用） |
| B. 动态生成推送 | 系统根据返工BOM自动组装数据推送金蝶 | 返工场景多变时（开发量大，后续考虑） |
