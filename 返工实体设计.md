# 返工实体设计

---

## 1. 返工单（计划层）

主单，由计划员发起。一张返工单可包含多个产品、多个批次。计划员在计划阶段按需分批下达，每次下达新建对应的返工生产单和申购单（草稿状态）；返工单审批通过后，草稿单据统一变为可执行状态。

**主表**

- 返工单号（自动生成）
- 计划开始时间
- 计划结束时间
- 状态（草稿 / 审批中 / 已审批 / 已驳回）
- 关联审批模板编号
- 创建人
- 创建时间

**明细子表**（每行对应一个物料+批次）

- 行号
- 物料编码（可选物料类型：成品 / 中间品 / 配方产品 / 基料）
- 物料名称（由物料编码带出）
- 批次ID（关联批次）
- 返工类型（外包装返工 / 内包装返工 / 配制返工）
- 返工数量
- 已下达生产数量（汇总，不超过返工数量）
- 关联BOM编号（计划员选定的具体BOM；BOM类型/版本号由此带出）
- 备注

---

## 2. 审批模板

定义审批链路，配置每一步由谁审批、以什么方式审批。审批模板可按单据类型复用。

- 模板编号（自动生成）
- 模板名称
- 适用单据类型（返工单 / 申购单 / 通用）
- 状态（启用 / 禁用）
- 


- 创建时间
- **审批步骤（子表）**
  - 步骤序号
  - 步骤名称
  - 审批角色 / 指定审批人
  - 审批方式（会签 / 或签）
  - 驳回规则（驳回至上一步 / 驳回至发起人）
  - 备注

---

## 3. 审批记录

记录某张单据发起审批后，每一步的实际审批情况。一张单据对应一条审批记录。

- 审批记录编号（自动生成）
- 关联单据类型（返工单 / 申购单）
- 关联单据编号
- 关联审批模板编号
- 当前步骤序号
- 整体状态（审批中 / 已通过 / 已驳回）
- 发起时间
- 完成时间
- **步骤审批明细（子表）**
  - 步骤序号
  - 步骤名称
  - 审批人
  - 状态（待审批 / 审批中 / 已通过 / 已驳回）
  - 审批时间
  - 审批结果（通过 / 驳回 / 转交）
  - 审批意见

> **生成时机**：审批发起时，按审批模板**批量预建所有步骤明细行**，第一步状态为「审批中」，其余步骤状态为「待审批」，审批时间/结果/意见为空。每步完成后：当前行填入时间/结果/意见并置为「已通过/已驳回」，下一步行状态改为「审批中」。驳回时，对应行重置为「审批中」，其后各行重置为「待审批」，已有的审批意见保留。

---

## 4. 返工生产单（执行层）

由计划员在计划阶段手动下达创建，初始为草稿状态；返工单审批通过后变为待领料，生产员可执行。同一张返工单可多次下达，每次新建一张，累计下达数量不超过返工单明细行的返工数量。

- 返工生产单号（自动生成）
- 关联返工单号
- 关联返工单明细行号
- 物料编码
- 物料名称
- 批次ID（关联批次）
- 返工类型
- 关联BOM编号
- 下达数量
- 状态（草稿 / 待领料 / 生产中 / 已完成 / 已关闭）
- 强制关闭标记（是 / 否）
- 强制关闭原因
- 强制关闭人
- 强制关闭时间
- 创建时间

---

## 5. 申购单

计划员下达生产时由系统同步创建，初始为草稿状态；返工单审批通过后变为待确认，申购员可执行。与返工生产单一一对应。

- 申购单号（自动生成）
- 关联返工单号
- 关联返工生产单号
- 工作流状态（草稿 / 待确认 / 已确认 / 已关闭）— 反映申购员操作节点
- 采购状态（未采购 / 部分采购 / 全部采购 / 无需采购）— 从明细行汇总
- 回料状态（未回料 / 部分已回 / 全部已回 / 无需回料）— 从明细行汇总
- 申购员
- 创建时间
- 关闭时间
- **申购明细（子表）**
  - 物料编码
  - 物料名称
  - 规格型号
  - 计划申购数量（系统按返工数量 × BOM用量自动计算）
  - 确认申购数量（申购员填写；可低于计划数量，例如计划需要 12、仓库已有 8，则填 4；填 0 表示本行无需申购）
  - 采购单ID（合并确认生成采购单后写入；一张申购单的不同行可能写入不同采购单ID）
  - 采购状态（未采购 / 采购中 / 已采购 / 无需采购）
  - 实际到货数量
  - 回料状态（未回料 / 已回料 / 无需回料）
  - 单位
  - 供应商
  - 预计到货时间
  - 实际到货时间
  - 备注

> **物料状态展示（界面查询，不存储）**：申购员在确认界面查看每行物料时，系统实时查询并展示以下数据供决策参考，均不落库：
> - 仓库库存数量
> - 申购中数量（其他申购单中已提交但未到货的数量）
> - 采购中数量（已下采购单但未到货的数量）
> - 生产中占用数量（当前返工/生产单已领料的数量）

---

## 6. 采购单

采购员在某张已确认的申购单中发起采购操作，系统同时展示其他已确认申购单中同一供应商的待采购物料，采购员勾选需要合并的物料后提交确认，系统此时生成采购单。同一物料来自多张申购单时，采购明细合并为一行（数量合计）；来源拆解通过"采购明细来源"关联子表追溯。

- 采购单号（自动生成）
- 供应商
- 采购员
- 状态（采购中 / 已完成 / 已取消）
- 创建时间
- 完成时间
- 备注
- **采购明细（子表）**：按物料合并，同一物料只有一行，数量为所有来源申购明细行的合计
  - 采购单号（关联主表）
  - 物料编码
  - 物料名称
  - 规格型号
  - 采购数量（合计）
  - 单位
  - 单价
  - 预计到货时间
  - 备注

> 采购单生成时，同步将所有关联的申购明细行（通过`申购明细.采购单ID`反查）`采购状态`置为"采购中"，并汇总更新对应申购单主表`采购状态`；采购完成时同步置为"已采购"。

---

## 7. 回料记录

物料到货后由仓库执行验收入库，记录实际回料情况。与采购明细行 1:1 对应（一条采购明细行对应一条回料记录）。申购单追溯通过`采购明细来源`子表反查，无需在此冗余存储。

- 回料单号（自动生成）
- 关联采购单号
- 关联采购明细行ID
- 状态（待验收 / 已完成）
- 入库仓库
- 库位
- 物料编码
- 物料名称
- 规格型号
- 应收数量（来自采购明细）
- 实收数量
- 单位
- 批次ID（关联批次）
- 质检结论（待检 / 合格 / 不合格）
- 不合格原因（质检结论为不合格时填写）
- 收料人
- 收料时间
- 备注

> 回料记录登记完成后：① 同步将对应申购明细行（经采购明细来源反查）的`回料状态`置为"已回料"、`实际到货数量`写入，并汇总更新申购单主表`回料状态`；② 自动触发生成一条**入库单**（入库类型 = 采购入库，来源单据类型 = 回料单，来源单据编号 = 回料单号），仓库入库审批通过后正式更新库存；③ 同步创建**批次记录**（来源类型 = 采购，来源单据编号 = 回料单号，初始批次状态 = 待检），质检结论填写后同步更新批次状态。

---

## 8. 物料

独立主数据，系统中所有涉及物料的单据（BOM、领料、申购、采购等）均通过物料编码关联此表，不重复存储物料基本信息。

- 物料ID（自动生成，主键）
- 物料编码（工厂料号，唯一）
- 物料名称
- 规格型号
- 单位
- 物料类型（原料 / 内包材 / 外包材 / 基料 / 配方产品 / 中间品 / 成品）
- 是否支持批次追溯（是 / 否）
- 默认供应商
- 状态（启用 / 禁用）
- 备注

---

## 9. 供应商

独立主数据，采购单、申购单、物料默认供应商均关联此表。

- 供应商ID（自动生成，主键）
- 供应商编码（唯一）
- 供应商名称
- 联系人
- 联系电话
- 地址
- 状态（启用 / 禁用）
- 备注

---

## 10. BOM

一个产品可以有多个 BOM 版本。BOM 与工艺路线 1:1，步骤直接作为 BOM 的子表存放，无需独立主表。物料与步骤之间为多对多关系（同一物料可在多个步骤中投入），通过子表三（步骤投料明细）关联，子表三同时记录各步骤的实际投料比。

**主表**

- ID（自动生成，主键）
- BOM编号（业务编号）
- BOM名称
- 所属物料编码
- 所属物料名称
- BOM类型（主BOM / 返工BOM）
- 版本号
- 状态（启用 / 禁用）
- 创建人
- 创建时间

**物料明细（子表一）**

- ID（自动生成，主键）
- BOMID（关联主表）
- 物料编码（工厂料号）
- 物料名称
- 版本号
- 供应商
- 是否支持追溯
- 配比%
- 最大版用量%
- 物料类型（原料 / 内包材 / 外包材 / 基料 / 配方产品 / 中间品 / 成品）
- 备注

**工艺步骤（子表二）**

- ID（自动生成，主键）
- BOMID（关联主表）
- 步骤序号
- 步骤描述（操作说明）
- 备注

**步骤参数配置（子表三）**

每个步骤可配置多条参数，不限参数类型（温度、压力、转速、pH 等均可）。

- ID（自动生成，主键）
- 工艺步骤ID（关联子表二）
- 参数名称（如：温度 / 压力 / 转速 / pH）
- 计划值
- 单位（如：℃ / MPa / r/min）
- 上限值（可选，超出则预警）
- 下限值（可选）
- 备注

**步骤投料明细（子表四）**

- 工艺步骤ID（关联子表二）
- 物料ID（关联子表一）
- 投料比%（该步骤的实际投料比，同一物料在不同步骤可不同）
- 备注

---

- 领料人
- 领料时间
- 物料编码
- 物料名称
- 规格型号
- 计划领料数量
- 实际领料数量
- 单位
- 批次ID（关联批次）
- 库位
- 备注

- 退料单号（自动生成，同一次退料操作共用同一单号）
- 关联返工生产单号
- 退料人
- 退料时间
- 物料编码
- 物料名称
- 规格型号
- 退料数量
- 单位

- 库位
- 退料原因
- 备注

## 13. 报废记录

归属于返工生产单，记录拆解产生的不可用物料。一次报废操作生成一个`报废单号`，同一单号下可有多行物料。

- 报废单号（自动生成，同一次报废操作共用同一单号）
- 关联返工生产单号
- 报废人
- 报废时间
- 物料编码
- 物料名称
- 规格型号
- 报废数量
- 单位
- 批次ID（关联批次）
- 库位
- 报废原因
- 备注

## 14. 领料单

返工生产单变为"待领料"状态时，系统依据关联BOM及下达数量自动生成，作为该生产单的备料与领料依据。一张返工生产单对应一张领料单。

- 领料单号（自动生成）
- 关联返工生产单号
- 状态（待领料 / 领料中 / 已完成）
- 创建时间
- 物料编码
- 物料名称
- 规格型号
- 计划领料数量（BOM用量 × 下达数量，系统自动计算）
- 单位
- 备注

> **汇总展示（界面查询，不存储）**：查看领料单时实时汇总，供生产员核对：
> - 已领数量（从领料记录汇总）
> - 已退数量（从退料记录汇总）
> - 已报废数量（从报废记录汇总）
> - 净消耗数量 = 已领 − 已退 − 已报废

---

## 15. 报损记录

生产完成时由生产员填写，记录本次生产的物料损耗及可回收情况。一张返工生产单对应一条报损记录。

- 报损单号（自动生成）
- 关联返工生产单号
- 报损人
- 报损时间
- 物料编码
- 物料名称
- 规格型号
- 报损数量
- 可回收数量
- 单位
- 备注

---

## 16. 工序执行记录

生产员领料完成后点击"开始生产"，生产单状态变为"生产中"时，系统依据关联 BOM 的工艺步骤**批量自动生成**，每步骤一行，状态初始为"待执行"。所有返工类型均生成此记录。步骤备料记录和步骤投料记录仅在**配制返工**时产生，外包装/内包装返工不产生。

- 关联返工生产单号
- 关联BOM工艺步骤ID
- 步骤序号
- 步骤描述（从BOM带出）
- 执行人
- 开始时间
- 完成时间
- 状态（待执行 / 执行中 / 已完成 / 异常）
- 异常说明
- 备注

**执行参数记录（子表）**

开始执行时从 BOM 步骤参数配置带出，执行中填写实际值。

- 关联工序执行记录ID
- 关联步骤参数配置ID
- 参数名称（从BOM带出）
- 计划值（从BOM带出）
- 实际值
- 单位（从BOM带出）
- 上限值 / 下限值（从BOM带出）
- 是否超限（系统自动判断）
- 备注

---

## 17. 步骤备料记录

每个工序执行前，记录该步骤所需物料的备料情况（称量/准备阶段）。与 BOM 步骤投料明细（子表三）关联，形成计划投料量对比。

- 关联返工生产单号
- 关联BOM工艺步骤ID
- 步骤序号
- 物料编码
- 物料名称
- 规格型号
- 计划备料数量（BOM投料比 × 下达数量，系统带出）
- 实际备料数量
- 批次ID（关联批次）
- 备料人
- 备料时间
- 备注

---

## 18. 步骤投料记录

每个工序执行中，记录实际投入该步骤的物料数量。与备料记录独立，反映真实投料情况。

- 关联返工生产单号
- 关联BOM工艺步骤ID
- 步骤序号
- 物料编码
- 物料名称
- 规格型号
- 实际投料数量
- 批次ID（关联批次）
- 投料人
- 投料时间
- 备注

---

## 19. 入库单

记录物料正式入库的库存事务，覆盖所有入库来源场景，是库存数量变更的唯一凭证。

**主表**

- 入库单号（自动生成）
- 入库类型（生产入库 / 采购入库 / 调拨入库 / 盘盈）
- 来源单据类型（返工生产单 / 回料单 / 手工）
- 来源单据编号
- 入库仓库
- 状态（待入库 / 已入库）
- 入库人
- 入库时间
- 备注

**入库明细（子表）**

- 行号
- 来源明细行ID（来源单据的具体明细行，可空）
- 物料编码
- 物料名称
- 规格型号
- 批次ID（关联批次）
- 库位
- 计划入库数量（来源明细行带出，可空）
- 实际入库数量
- 单位
- 关联质检单号（可空）
- 备注

> **触发说明**：① 生产员在返工生产单发起入库操作时，系统创建入库单（入库类型 = 生产入库，来源单据类型 = 返工生产单）；② 回料记录验收完成后，系统自动创建入库单（入库类型 = 采购入库，来源单据类型 = 回料单）。入库单审批通过后，根据明细行更新`库存`表。

- 关联领料单号
- 关联返工生产单号
- 状态（待审批 / 已通过 / 已驳回）

- 审批时间
- 审批意见
- 备注

## 21. 仓库退料审批

生产员发起退料后，仓库对退料动作进行审批确认。与退料记录 1:1 对应。

- 审批单号（自动生成）
- 关联退料单号
- 关联返工生产单号
- 状态（待审批 / 已通过 / 已驳回）
- 审批人
- 审批时间
- 审批意见
- 备注

---

## 22. 仓库报废确认

生产员登记报废后，仓库对报废物料进行确认接收。与报废记录 1:1 对应。

- 确认单号（自动生成）
- 关联报废单号
- 关联返工生产单号
- 状态（待确认 / 已确认）
- 确认人
- 确认时间
- 备注

---

## 23. 仓库入库审批

入库单提交后，仓库对入库动作进行审批确认。与入库单 1:1 对应。

- 审批单号（自动生成）
- 关联入库单号
- 状态（待审批 / 已通过 / 已驳回）
- 审批人
- 审批时间
- 审批意见
- 备注

---

## 24. 用户

系统操作人主数据，所有单据的操作人、审批人均关联此表。

- 用户ID（自动生成，主键）
- 用户编码（登录账号，唯一）
- 姓名
- 所属部门ID
- 角色（计划员 / 生产员 / 申购员 / 采购员 / 仓库 / 审批人 / 管理员，可多选）
- 状态（启用 / 禁用）
- 备注

---

## 25. 部门

组织架构主数据，用户归属部门。

- 部门ID（自动生成，主键）
- 部门编码（唯一）
- 部门名称
- 上级部门ID（支持多级，根部门为空）
- 状态（启用 / 禁用）
- 备注

---

## 26. 库存

记录各仓库中每种物料每个批次在各库位的当前库存数量。申购单确认界面"仓库库存数量"实时查询此表。

- 物料编码
- 仓库
- 库位
- 批次ID（关联批次）
- 库存数量
- 单位
- 最后更新时间

> **主键说明**：`(物料编码, 仓库, 库位, 批次ID)` 四元组唯一标识一条库存记录，同一批次在不同库位视为独立记录。

---

## 27. 库位

仓库内的存储位置主数据，按仓库 → 库区 → 排 → 列 → 层五级定位，唯一确定物理存放点。

- 库位ID（自动生成，主键）
- 库位编码（唯一，建议按坐标规则生成，如 A-01-03-02）
- 库位名称
- 所属仓库
- 库区（成品区 / 原料区 / 待检区 / 隔离区）
- 排号（货架排编号，如 01、02）
- 列号（货架列编号，如 01、02）
- 层号（货架层编号，如 01、02）
- 状态（启用 / 禁用）
- 备注

---

## 28. 批次

物料批次主数据，唯一标识一批物料的来源、质量状态与有效期。批次与物料关联，是库存、领料、入库等所有操作的追溯锚点。

- 批次ID（自动生成，主键）
- 批次号（业务编号，同一物料下唯一）
- 物料编码（关联物料）
- 生产日期
- 有效期 / 失效日期
- 批次状态（计划中 / 待检 / 合格 / 不合格 / 冻结 / 已耗尽）
- 来源类型（自制生产 / 采购 / 委外加工）
- 来源单据编号（关联回料单号 / 生产单号）
- 供应商批次号（可空，采购批适用）
- 父批次ID（可空，拆批场景适用）
- 创建时间
- 备注

> **创建时机**：采购物料在**回料记录登记时**由系统自动创建（状态 = 待检）；自制生产批次在**生产单下达时**预建（状态 = 计划中）；返工不新建批次，沿用原批次。批次状态随质检结论自动更新。

---

## 29. 质检

记录返工生产单的质量检验情况，由生产员手动发起，质检员执行填写。

- 质检单号（自动生成）
- 关联返工生产单号
- 质检员
- 质检时间
- 质检结论（待检 / 合格 / 不合格）
- 不合格项说明
- 处理建议（返工 / 报废 / 放行）
- 备注

## 实体关系简图

```
【主数据】
部门
用户（关联部门）

审批模板
  └── 审批步骤

物料

供应商

库位（所属仓库）

批次（物料 × 批次号）

库存（物料 × 仓库 × 库位 × 批次）

BOM
  ├── 物料明细
  ├── 工艺步骤
  │    └── 步骤参数配置
  └── 步骤投料明细

【业务流】
返工单
  ├── 明细子表
  ├── 关联审批模板 ──→ 发起审批 ──→ 审批记录
  │                                   └── 步骤审批明细
  │
  ├── 计划员分批下达 → 返工生产单
  │    ├── 领料单
  │    ├── 领料记录 ──→ 仓库领料审批
  │    ├── 退料记录 ──→ 仓库退料审批
  │    ├── 报废记录 ──→ 仓库报废确认
  │    ├── 工序执行记录
  │    │    ├── 执行参数记录
  │    │    ├── 步骤备料记录
  │    │    └── 步骤投料记录
  │    ├── 报损记录
  │    └── 入库单（生产入库）──→ 仓库入库审批
  │
  └── 计划员下达时同步创建 → 申购单
       ├── 申购明细
       ├── 关联审批模板 ──→ 发起审批 ──→ 审批记录
       │                                   └── 步骤审批明细
       └── 申购员确认 → 采购员合并 → 采购单
                                      ├── 采购明细
                                      │    └── 采购明细来源
                                      └── 物料到货 → 回料记录 ──→ 入库单（采购入库）──→ 仓库入库审批
```
