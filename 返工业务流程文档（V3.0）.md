# 化妆品成品返工业务流程文档（V3.0）

> **版本说明**：本版本基于实际业务梳理重新整理，聚焦系统功能边界与操作节点，明确各角色职责。

---

## 一、总体流程概览

返工流程分为**两条并行线**，均由审批节点触发：

```
                     ┌─────────────────────────────────────────┐
                     │           计划员 发起创建返工单            │
                     │  选产品批次 → 配置BOM+工艺路线 → 确认需求  │
                     └──────────────────┬──────────────────────┘
                                        │
                                     审  批
                                        │
               ┌────────────────────────┴────────────────────────┐
               │                                                  │
               ▼                                                  ▼
   【返工生产单（按产品/批次拆分）】                        【申购单】
   领料 → 退料/报废                               申购员按实际情况申购
   ↓                                             → 采购 → 回料
   生产员按工艺路线执行
   ↓
   手动发起后续节点（质检/灌装/包装/入库）
   ↓
   完成 / 强制关闭
```

---

## 二、阶段一：计划发起 — 创建返工单

### 2.1 操作入口

由**计划员**手动发起，创建返工单。

### 2.2 填写基本信息

| 字段 | 说明 | 系统校验/动作 |
|------|------|-------------|
| 产品 | 选择成品 | — |
| 产品批次 | 选择已入库批次 | **自动校验库存是否存在**，无库存则拦截 |
| 返工类型 | 外包装返工 / 内包装返工 / 配制返工 | 影响后续BOM推荐和工艺路线范围 |
| 计划开始时间 | 计划员填写 | — |
| 计划结束时间 | 计划员填写 | — |
| 返工数量 | 本次返工的数量 | — |

### 2.3 BOM 配置

**BOM 说明**：BOM 结构一般为：成品 = 产品本体 + 所需原料 / 内包材 / 外包材。

**操作流程**：

```
系统自动带出主BOM（该产品的默认BOM）
         │
         ▼
计划员选择使用哪个BOM（可选方式如下）：
  ┌─── 方式A：从现有BOM中选择一个返工适用的BOM
  └─── 方式B：跳转BOM管理页面，新建返工专用BOM
                  ↓
         在BOM页面同步配置工艺路线
```

> **注意**：BOM 与工艺路线是绑定的，在BOM页面一并配置完成后回到返工单关联。

### 2.4 工艺路线说明

工艺路线描述的是**本次返工涉及的不合格步骤的具体操作方法**，例如：
- 外包装返工：说明外包装这步怎么做（如贴标顺序、标签对齐要求）
- 内包装返工：说明灌装这步怎么做（如灌装量标准、封口要求）
- 配制返工：说明配制这步怎么做（如回锅温度、调整用料加入时机）

**重要约束**：

> 工艺路线**只覆盖错误/不合格的操作步骤**，指导生产员如何纠正。  
> 工艺路线之后的流程（如质检、灌装、包装、入库）**由生产员在系统中手动发起**，系统仅开放入口，不自动流转。

**示例 — 内包装返工的工艺路线范围**：

```
工艺路线说明：灌装步骤怎么做
                 ↓
（以下不在工艺路线中，由生产员手动发起）
         → 包装
         → 成品质检
         → 入库
```

### 2.5 下达生产和申购

BOM 配置完成后，计划员按需分批下达：

- 每次指定本次下达数量，系统新建返工生产单和申购单（草稿状态）
- 申购明细根据返工数量 × BOM 物料用量自动计算
- 可多次下达，每次新建单据，累计下达数量不得超过返工单明细行的返工数量

---

## 三、阶段二：审批

计划员提交后，进入审批流程。

| 审批结果 | 系统动作 |
|---------|---------|
| **通过** | 计划阶段已创建的**返工生产单**和**申购单**（草稿）统一变为可执行状态 |
| **驳回** | 返回计划员修改，重新提交 |

---

## 四、阶段三：返工生产单执行

### 4.1 单据创建规则

返工生产单由计划员在计划阶段手动下达创建，审批通过后状态变为待领料。同一张返工单明细行可多次下达，每次新建一张返工生产单，累计下达数量不超过该行返工数量。

### 4.2 领料

生产员按返工BOM领取所需物料。

### 4.3 退料 / 报废

领料后，可在返工生产单中发起：
- **退料**：将多领或不需要的物料退回仓库
- **报废料**：将拆解产生的不可用物料进行报废登记

### 4.4 按工艺路线执行生产

生产员参照返工BOM中关联的工艺路线，完成对应的生产操作步骤。

### 4.5 手动发起后续节点

工艺路线覆盖的步骤完成后，生产员（操作员）在系统中**手动发起后续节点**，系统仅开放入口，不做自动流转：

| 可手动发起的节点 | 说明 |
|---------------|------|
| 质检 | 发起质量检验 |
| 灌装 | 发起灌装操作记录 |
| 包装 | 发起包装操作记录 |
| 入库 | 发起成品入库 |

> 操作员根据实际生产进度，按需选择发起哪个节点，不强制顺序流转。

### 4.6 强制关闭

支持对返工生产单**强制关闭**，适用于：
- 返工方案变更导致原单废弃
- 决定直接报废不再返工
- 其他异常情况需要终止执行

强制关闭后，单据状态置为"已关闭"，不可继续操作。

---

## 五、阶段四：申购单执行

申购单由计划员下达生产时同步创建（草稿），审批通过后变为待确认，与对应返工生产单一一对应。

```
计划阶段创建（草稿）→ 审批通过 → 待确认
     ↓
申购员查看申购单：
  - 系统实时查询并展示每行物料的多维状态（不存储）：
      仓库库存数量 / 申购中数量 / 采购中数量 / 生产中占用数量
  - 申购员参考以上数据，逐行填写【确认申购数量】（可低于计划数量，也可填 0）
  - 填写完成后提交确认
     ↓
     ├─── 存在确认申购数量 > 0 的行 ───→ 已确认 → 采购员执行采购
     │                                          ↓
     │                                   物料到货入库（回料）
     │                                          ↓
     │                                      已关闭
     │
     └─── 所有行确认申购数量均为 0 ──→ 直接已关闭（无需采购）
```

> **说明**：申购员在确认界面可以看到每种物料的当前库存数量，若库存已满足生产需求，可将对应行的确认申购数量填为 0，无需走采购流程。

| 角色 | 职责 |
|------|------|
| 申购员 | 审查申购需求，查看库存数量，逐行决定是否申购及申购数量 |
| 采购员 | 执行对外采购（仅当有确认申购数量 > 0 时介入） |
| 仓库 | 验收到货，执行回料入库 |

## 六、角色职责汇总

| 角色 | 参与阶段 | 主要职责 |
|------|---------|---------|
| **计划员** | 发起阶段 | 创建返工单、配置BOM+工艺路线、确认生产/申购需求 |
| **审批人** | 审批阶段 | 审核返工方案，通过/驳回 |
| **生产员/操作员** | 生产执行阶段 | 领料、退料/报废、按工艺路线生产、手动发起后续节点 |
| **申购员** | 申购执行阶段 | 审查申购需求，决策是否申购 |
| **采购员** | 申购执行阶段 | 执行外部采购 |
| **仓库** | 申购执行阶段 | 验收到货，执行回料入库 |

## 七、关键设计约束

1. **库存校验前置**：选择产品批次时，系统实时校验库存是否存在，无库存直接拦截，不允许创建。

2. **BOM与工艺路线绑定**：工艺路线在BOM维度配置，两者不拆分管理。计划员可新建返工专用BOM（与工艺路线一起在BOM页面完成配置），也可复用已有BOM。

3. **工艺路线只覆盖不合格步骤**：工艺路线不描述完整生产流程，只说明"哪个步骤出了问题，该步骤如何操作"。后续质检、灌装、包装、入库等节点系统只开入口，由操作员手动发起。

4. **单据按产品+批次拆分**：一次返工事件可涉及多个产品或多个批次，审批通过后系统自动按"一产品一批次"拆分为多张独立的返工生产单执行。

5. **两条独立执行线**：返工生产单（执行返工操作）与申购单（采购补充物料）审批后并行，互不阻塞。

6. **强制关闭能力**：任意状态下均可强制关闭返工生产单，用于应对异常情况或决策变更。

## 八、状态流转

### 8.1 返工单（计划层）

```
草稿 → 审批中 → 已审批（草稿生产单和申购单变为可执行）
                    ↑
              驳回 → 退回草稿
```

### 8.2 返工生产单（执行层）

```
草稿（计划阶段）→ 待领料（审批通过）→ 生产中（领料完成后）→ 已完成（入库后）
                                                               │
                                                     强制关闭 → 已关闭（任意阶段可操作）
```

### 8.3 申购单

申购单状态分三个维度独立跟踪：

**工作流状态**（申购员操作节点）
```
草稿（计划阶段）→ 待确认（审批通过）→ 已确认（申购员提交确认后）→ 已关闭
```

**采购状态**（按明细行汇总，自动更新）
```
无需采购（所有行确认申购数量 = 0）
未采购   （有行需采购，但尚未下采购单）
部分采购 （部分行已有采购单）
全部采购 （所有需采购行均已采购）
```

**回料状态**（按明细行汇总，自动更新）
```
无需回料 （所有行无需采购）
未回料   （有到货行但均未回料）
部分已回 （部分行已回料）
全部已回 （所有需回料行均已到货入库）
```

**明细行级状态**（每行独立）
```
采购状态：无需采购 / 未采购 / 采购中 / 已采购
回料状态：无需回料 / 未回料 / 已回料
```
